# 项目计划：蓝桥杯单片机 AI 辅助学习助手

## 一、项目目标

本项目计划构建一个面向蓝桥杯单片机 C 组备赛学生的 AI 辅助学习工具。

项目主要围绕 STC15F2K60S2 单片机常见模块，整理示例代码、提示词模板和学习文档，并计划后续接入 Xiaomi MiMo API，用于辅助代码解释、错误排查、学习笔记生成和训练题生成。

## 二、使用场景

### 1. 单片机代码解释

学生在学习蓝桥杯单片机代码时，经常会遇到以下问题：

- 不理解函数之间的调用关系
- 不清楚全局变量的作用
- 不知道定时器中断如何刷新数码管
- 看不懂 DS18B20 和 IIC 通信流程
- 不知道按键、显示、LED 如何组合成完整功能题

因此，本项目希望通过大模型辅助解释代码，让初学者能更快理解程序逻辑。

### 2. 错误排查

蓝桥杯单片机项目中常见问题包括：

- 数码管不亮或显示错乱
- 按键扫描无反应
- 温度读取结果异常
- IIC 读取值一直为 0 或 255
- LED 状态和题目要求不一致
- 定时器中断影响外设通信

项目计划使用 AI 根据代码和现象给出调试思路，帮助学生缩小排查范围。

### 3. 学习笔记整理

在备赛过程中，学生需要反复复盘不同模块。  
本项目计划使用 AI 自动整理模块学习笔记，包括：

- 模块功能
- 代码结构
- 常见错误
- 调试步骤
- 考试中可能的变形题

### 4. 训练题生成

项目后续计划根据蓝桥杯常见模块生成训练题，例如：

- 按键切换显示界面
- 温度采集与显示
- 参数设置与保存
- ADC 采集与 DAC 输出
- LED 联动控制
- 继电器与蜂鸣器控制

## 四、技术需求

项目后续希望接入 Xiaomi MiMo API，主要需要以下能力：

### 1. 长上下文代码理解

单片机项目通常不是单文件代码，可能包含：

- `main.c`
- `iic.c`
- `iic.h`
- `onewire.c`
- `onewire.h`
- 显示模块
- 按键模块
- 定时器模块

因此需要模型能理解多个文件之间的调用关系。

### 2. C51 代码分析能力

项目代码主要基于 C51 和 STC15F2K60S2，需要模型能理解：

- 寄存器配置
- 定时器中断
- P0 / P2 端口控制
- 74HC573 锁存器
- 数码管动态扫描
- 外设通信流程

### 3. 调试推理能力

项目希望模型不仅能解释代码，还能根据运行现象分析可能原因，例如：

- 是代码逻辑错误
- 是定时器刷新问题
- 是外设通信问题
- 是端口初始化问题
- 是硬件连接或模块依赖问题

### 4. 低成本持续调用能力

备赛学习中需要频繁分析代码、生成笔记和排查错误，因此需要稳定、低成本的大模型 API 支持。
